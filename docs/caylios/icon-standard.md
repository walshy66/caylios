# Caylios Icon Construction Standard — Developer Contract

> Source: `Icon Construction Standard_V1.0.docx` (V1.0 FINAL, distributed to Directors).
> The .docx is the human-authored source of truth; update this file when a new version lands.

## Scope

Governs icon **construction** only: new icon creation, modification of existing icons, expansion of the Icon Library, and third-party designers creating Caylios icons. Icon usage and meaning are owned by the [Visual Language Guide](visual-language.md); approved icons live in the Icon Library.

Governance chain: Brand Kit (identity) → Visual Language Guide (usage) → Icon Construction Standard (construction) → Icon Library (approved icons).

## Core principle

Leverage established visual conventions wherever practical. Caylios identity comes from consistent construction and application, not reinvented metaphors. Consistency takes precedence over individual creativity; recognition takes precedence over originality.

## Construction spec

- **Grid:** 24×24 px master grid, 2 px safe area, primary geometry on whole-pixel coordinates, pixel snapping for raster export. No alternatives.
- **Stroke:** single 2 px stroke, rounded caps, rounded joins, maximum corner radius 2 px.
- **Geometry:** true circles; consistent corner radius on rectangles; horizontal/vertical/45° diagonals; irregular angles only when essential to recognition; optical balance over mathematical symmetry where required.
- **Detail:** minimum visual information necessary. No shading, gradients, perspective, 3D, or shadows.
- **Visual weight:** light and consistent; generous negative space; icon occupies ~70–75% of the usable construction area; similar concepts get similar density.
- **Abstraction:** symbols over realistic objects; one concept per icon; no decorative interpretation; consistent abstraction level across the library.
- **Colour:** normally one approved brand colour; status colours only for operational state; colour never alters construction or meaning, and is never the sole carrier of meaning.
- **Scaling:** 16 px may use the simplified icon to preserve recognition; 20–24 px standard construction; 32 px+ standard construction without increased stroke weight.

## Construction sequence

1. Select the recognised visual convention (only invent a new symbol when no convention exists, an existing one is ambiguous, or a new symbol measurably improves understanding).
2. Build the primary geometry.
3. Apply the approved stroke rules.
4. Adjust for optical balance.
5. Remove unnecessary detail.
6. Validate against this standard.
7. Apply approved colour.

## Prohibitions

Icons must not: mix stroke weights or widths, use inconsistent corner radii, distort/stretch/compress proportions, exceed approved density, combine multiple concepts, invent metaphors where conventions exist, use filled shapes unless defined by the Icon Library, add decorative detail, perspective, shadows, or gradients, or rely on colour alone for meaning.

## Quality check (before approval)

1. Based on a recognised convention where appropriate?
2. Communicates a single concept?
3. Immediately recognisable?
4. Legible at 16 px?
5. Follows the construction grid?
6. Maintains approved visual weight?
7. No removable detail without losing meaning?
8. Maintains approved abstraction level?
9. Feels like part of the existing library?

When strict representation and library consistency conflict, consistency wins. Expand the library by extending the existing construction language, never redefining it.
